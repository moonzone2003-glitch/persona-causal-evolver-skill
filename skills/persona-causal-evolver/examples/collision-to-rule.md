# Example: 从通用项目复盘到候选人格规则

## 输入

- 某团队连续三次在项目交接时遗漏关键上下文，导致新接手成员重复排查同一问题。
- 用户推动团队把交接记录、问题触发、处理动作和验证结果整理成固定模板。
- 之后两次项目复盘中，用户继续要求将模板转成检查清单，而不是只写口头总结。

## 输出

- 触发：项目交接反复遗漏上下文，造成重复排查和协作成本上升。
- 行为：建立交接模板、要求记录触发条件和验证结果、推动模板检查清单化。
- 动机假设：希望降低重复错误，减少隐性上下文丢失，让团队协作可交接、可验证。
- 人格推断：可能偏好系统化、规则化和可复用流程。
- 证据等级：中。已有多次事件和产物，但仍需更多独立场景验证。
- 建议：暂存为候选人格规则，不直接写回真实人格；下一轮继续观察是否在不同领域重复出现。
- 反证条件：如果用户只在该项目被要求时这么做，在其他自主场景中并不主动系统化，则不能推断为稳定人格。
