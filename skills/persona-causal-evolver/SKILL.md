# Persona Causal Evolver

> 从“我做了什么”升级到“我为什么这么做、什么情况下会重复发生、是否值得写回人格模型”。

## 使用时机

- 用户已经有一个初始自我 Skill，希望持续升级它。
- 用户希望分析真实事件、对话碰撞、项目产物、方法论抽象背后的人格信号。
- 用户希望从 Agent 的各类文档、报告、产物、checkpoint、日志、训练记录中提取人格证据。
- 用户说“帮我判断这个行为说明什么”“这是不是我的稳定模式”“要不要更新到我的人格 Skill”。
- 用户希望把多轮讨论产物、复盘记录、训练结果转成可验证的人格规则。
- 用户希望判断训练后的升维我/专家对照结果，是否已经足够稳定，可以回灌真实人格 Skill。

## 主分类

| 项 | 说明 |
|---|---|
| 主分类 | 数据获取和分析 |
| 不做 | 不做初始人格蒸馏；不替代训练计划；不把模拟高分版直接写成真实人格 |
| 关联技能 | `yourself-skill` 负责初始蒸馏；`self-growth-coach` 负责训练、评分、复盘；个人自我 Skill 负责承载已验证人格 |

## 输入与输出

| 项 | 说明 |
|---|---|
| 输入 | 事件、对话摘要、Agent 文档、项目产物、checkpoint、报告、复盘记录、训练日志、候选策略人格、已有自我 Skill |
| 输出 | 因果样本、证据等级、候选人格规则、回灌建议、训练建议入口、是否可回灌裁决 |
| 依赖 | 文件读取、历史搜索、可选的表格/JSON/Markdown 日志；初级蒸馏依赖外部 `yourself-skill` |
| 状态 | 建议保存 append-only 因果样本，不直接覆盖人格 Skill |

## 核心边界

- 真实人格 Skill 只保存被多次证据验证的稳定模式。
- 本 Skill 保存和分析“候选人格规则”，不直接宣布用户已经改变。
- 训练后的高分版人格是策略模拟器，不是真实人格；必须经过现实行为验证后才能回灌。
- 自动化运行、单次任务、偶然情绪只能作为弱证据。
- 多轮沟通碰撞形成的产品、方法论、Skill、决策修正链，是高价值人格样本。
- 不保存、不回显原始敏感材料；只保留摘要、证据指针、证据等级、反证条件和候选规则。

## 执行流程

### Step 1: 收集人格证据

- 输入：用户给出的事件或待分析时间段。
- 动作：收集对话、产物、文件、复盘和训练记录，按来源标注可信度。
- 输出：证据清单和缺口。
- 失败：证据少于 3 条时，只能生成低可信度观察，不得写入人格规则。

### 🔴 CHECKPOINT 1 · 证据确认

进入因果推断前，必须展示：
- 哪些是事实证据。
- 哪些是推断。
- 哪些来源只适合作为弱证据。
- 是否存在数据缺口或反例。

### Step 2: 构造因果链

- 输入：证据清单。
- 动作：按 `触发 → 行为 → 动机假设 → 人格推断 → 理想自我 → 反证条件` 建模。
- 输出：一组因果样本。
- 失败：如果只能描述行为，不能解释动机，标注“动机证据不足”。

### Step 3: 判断证据等级

- 输入：因果样本。
- 动作：按高/中/低/噪声四级分类。
- 输出：人格证据等级。
- 失败：如果样本来自单次自动化、模板化日志、无互动产物，默认降级为低证据。

### Step 4: 生成候选人格规则

- 输入：中高证据样本。
- 动作：抽象为候选规则，写清适用场景、触发条件、行为倾向、反证条件。
- 输出：候选人格规则列表。
- 失败：没有反证条件的规则不能回灌，只能保留为观察。

### 🔴 CHECKPOINT 2 · 回灌确认

写入任何自我 Skill 前，必须暂停确认：
- 将写入哪个人格 Skill。
- 写入的是稳定人格、偏好、反模式，还是训练目标。
- 是否有足够现实行为证据。
- 是否会覆盖旧规则。

### Step 5: 回灌或转训练

- 输入：候选人格规则和用户确认。
- 动作：
  - 已验证稳定规则 → 建议写回自我 Skill。
  - 未验证但有价值 → 转给 `self-growth-coach` 设计训练任务。
  - 噪声或偶发行为 → 只记录，不回灌。
- 输出：回灌清单、训练清单、暂存清单。
- 失败：如目标人格 Skill 不存在，先输出 patch 建议，不直接创建个人模型。

### Step 6: 接收训练结果并裁决回灌

- 输入：`self-growth-coach` 输出的内化分、现实执行证据、延迟保持证据、候选升维我。
- 动作：检查训练结果是否只是模型输出变好，还是现实行为也发生稳定变化。
- 输出：`write_to_persona`、`keep_as_candidate`、`send_back_to_training` 三种裁决之一。
- 失败：如果缺少现实执行证据或延迟保持证据，默认 `keep_as_candidate`。

## 证据等级

| 等级 | 判定标准 | 可做什么 |
|---|---|---|
| 高 | 多轮对话、多次项目、明确产物、方向修正链一致 | 可生成候选人格规则 |
| 中 | 有明确事件和产物，但次数不足或缺反例 | 可暂存观察，等待验证 |
| 低 | 单次自动化、单次情绪、模板日志、弱上下文 | 只能作为辅助线索 |
| 噪声 | 无来源、无行为、无触发、只有评价词 | 丢弃 |

## 回灌规则

| 目标 | 写入条件 | 示例 |
|---|---|---|
| 真实人格 | 至少 2-3 个独立事件支持，有反证条件 | “倾向把反复问题抽象成规则/Skill/检查清单” |
| 偏好 | 用户明确表达或多次稳定选择 | “偏好结构化看板和可追溯索引” |
| 反模式 | 多次导致错误、返工或用户纠正 | “把行为策略误写成动机” |
| 训练目标 | 现实能力尚未稳定，只是希望提升 | “决策前先做反证和退出条件” |

## 与训练系统的关系

```text
初始自我 Skill
  ↓
真实事件 / 对话碰撞 / 产物 / 训练日志
  ↓
persona-causal-evolver 判断人格因果和证据等级
  ↓
稳定规则 → 回灌真实人格 Skill
未稳定目标 → 交给 self-growth-coach 训练
训练结果 → 回到 evolver 再判断是否稳定
```

## 失败模式与 fallback

| 触发条件 | 一线修复 | 仍失败兜底 |
|---|---|---|
| 证据不足 | 降级为观察，不输出人格结论 | 向用户要更多事件或产物 |
| 动机不清 | 标注多个假设和缺口 | 只保留行为样本 |
| 训练目标被误当真实人格 | 改为训练候选，不回灌 | 等待现实验证 |
| 自动化记录过多 | 降低自动化权重 | 只保留人工互动样本 |
| 规则互相冲突 | 展示冲突证据 | 暂停写入，等待用户裁决 |
| 训练结果只提升了模型回答 | 说明还没内化 | 保留为候选策略，不回灌 |
| 训练结果有执行但缺少延迟保持 | 可能只是短期模仿 | 继续观察并要求延迟复测 |
| 目标 Skill 不存在 | 输出建议结构 | 不擅自创建个人模型 |

## 反例与黑名单

| 不要做 | 原因 | 替代做法 |
|---|---|---|
| 把“沉淀为资产”写成动机 | 这是行为策略，不是内在动机 | 追问触发异常、风险、防御对象 |
| 把高分版自己写回真实人格 | 会制造能力幻觉 | 作为训练目标暂存 |
| 用单次行为定义人格 | 容易误判 | 至少要求多次证据或明确用户确认 |
| 把自动化产物当强证据 | 自动化只说明系统运行，不等于人格选择 | 降级为弱证据 |
| 只写结论不留证据 | 后续无法追溯 | 每条规则绑定来源和反证条件 |
| 覆盖旧人格规则 | 会丢失历史 | 追加候选、保留版本、确认后合并 |

## 输出模板

```text
结论：{是否值得更新人格模型}。

1. 证据清单
2. 因果链
3. 证据等级
4. 候选人格规则
5. 回灌/训练/暂存建议
6. 反证条件
```

## 验证方式

- 用 2 个历史项目产物样本测试：是否能区分行为、动机、人格推断。
- 用 1 个训练目标样本测试：是否拒绝直接写回真实人格。
- 检查每条候选规则是否有证据来源和反证条件。
- 检查是否能正确调用 `self-growth-coach` 作为训练后续，而不是自己承担训练。
- 用 1 个训练结果样本测试：是否能区分模型输出提升、现实执行、延迟保持和回灌裁决。
