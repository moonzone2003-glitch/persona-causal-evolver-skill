# 因果模型参考

## 分层

| 层 | 含义 | 例子 |
|---|---|---|
| 触发 | 事件如何启动 | 上下文丢失、规则被破坏、方向被修正 |
| 行为 | 可观察动作 | 更新 checkpoint、创建 Skill、写日报、整理索引 |
| 动机假设 | 为什么会这样做 | 想修复异常、降低重复错误、保持可交接 |
| 人格推断 | 稳定倾向 | 偏好抽象、方法论化、规则化 |
| 理想自我 | 想成为什么样的人 | 更稳、更有纪律、更能闭环 |
| 回灌结论 | 是否写回真实人格 | 仅限高证据稳定规则 |

## 证据等级

| 等级 | 标准 | 处理方式 |
|---|---|---|
| 高 | 多轮对话 + 多次产物 + 方向修正链一致 | 可提炼人格规则 |
| 中 | 有事件和产物，但次数不够 | 暂存观察 |
| 低 | 单次自动化、单次日志、弱上下文 | 只作弱证据 |
| 噪声 | 无来源、无行为、只有评价词 | 丢弃 |

## 回灌标准

- 需要至少 2-3 个独立证据。
- 必须写清反证条件。
- 不能把训练目标直接当稳定人格。
- 自动化运行只可做辅助证据。
