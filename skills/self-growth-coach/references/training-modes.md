# 训练模式

## 模式 A：每日复盘

目标：把当天真实行为转成训练数据。

流程：
1. 收集 3-5 个关键行为。
2. 标注领域：投资、学习、产品、管理、执行、AI Coding。
3. 每个行为记录：动作、意图、结果、证据、情绪、偏差。
4. 用六维通用评分打分。
5. 只生成 1-3 个明天能执行的训练任务。

输出：日报式复盘、分项评分、最小训练任务。

## 模式 B：专项训练

目标：针对一个短板做刻意练习。

流程：
1. 选择短板维度。
2. 生成 3 个难度递增场景。
3. 用户先作答。
4. 高分版自己给出对照答案。
5. 抽取差距规则。
6. 形成下一次训练题。

输出：训练题、用户答案评分、高分版答案、差距规则。

## 模式 C：高分版推演

目标：用 8分/10分目标态反推当前动作。

流程：
1. 明确当前场景。
2. 写出 5分我会怎么理解和行动。
3. 写出 8分我会怎么理解和行动。
4. 写出 10分我会怎么理解和行动。
5. 提炼唯一最关键差距。
6. 转成下一步动作。

输出：5/8/10 对照表、关键差距、下一步动作。

## 模式 D：月度进化

目标：判断真实能力是否提升，而不是只感觉变强。

流程：
1. 汇总本月评分日志。
2. 找出重复短板和已消失短板。
3. 对比月初目标态。
4. 更新能力分数和训练计划。
5. 判断是否需要更新自我 Skill 或本 Skill。

输出：月度成长报告、能力曲线、Skill 进化建议。

## 模式 E：知识内化

目标：把书、论文、文章、专家方法论转成用户自己的决策规则，而不是生成通用读书报告。

流程：
1. 明确用户当前短板或训练领域。
2. 从材料中抽取和短板相关的 1-3 条可迁移规则。
3. 标注哪些规则不适配用户现实约束。
4. 设计一个新场景迁移题。
5. 生成一个现实执行任务。
6. 用理解、迁移、决策改善、执行、保持五项评分。

输出：可迁移规则、用户约束过滤、迁移题、执行任务、内化评分。

## 模式 F：专家人格对照

目标：用专家标准拔高判断，但不让专家替代用户本人。

流程：
1. 写出当前我在场景中的判断。
2. 写出专家标杆判断。
3. 过滤掉不适配用户约束的专家建议。
4. 合成升维我判断。
5. 提炼唯一核心差距。
6. 生成一个可验证训练动作。

输出：当前我/专家/升维我三方对照、差距归因、训练任务。

## 模式 G：内化验证

目标：判断训练后是否真的变成用户能力，而不是只有模型输出更高级。

流程：
1. 收集训练前答案、训练后答案和现实执行证据。
2. 检查是否有新场景迁移。
3. 检查是否有延迟保持证据。
4. 调用 `scripts/score_internalization.py` 或手动按五项评分。
5. 判断是否交给 `persona-causal-evolver` 做回灌裁决。

输出：内化评分、证据缺口、候选策略状态、回灌裁决入口。
