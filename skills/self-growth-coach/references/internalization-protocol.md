# 内化与反哺协议

## 定位

Self Growth Coach 的产品定位是个人决策训练器：把真实决策样本转成当前我、专家标杆、升维我、训练任务、内化评分和回流门禁。

不要把它定位成泛化的成长陪聊。切入点是高频使用 AI、但在投资、AI Coding、学习、产品、管理等场景反复出现同类判断损失的人。

## 三种角色

| 角色 | 回答的问题 | 产物 | 风险 |
|---|---|---|---|
| 直接读数据的 Skill | 发生了什么，有哪些证据 | 样本、规律、缺口 | 只分析，不负责训练 |
| 专家人格 | 什么叫高水平判断 | 标杆答案、检查维度 | 脱离用户约束，正确但不可执行 |
| 升维我 | 在我的约束下，下次该怎么判断 | 候选策略、训练动作、回流候选 | 被误当成真实能力 |

升维我不是专家，也不是当前我。它是：

```text
当前我稳定特征 + 专家标准 + 学习材料中的规则 + 现实约束 + 反模式修正
```

## 训练模式

### 1. 场景重放

适用：用户给出真实投资、代码、产品、管理、学习决策。

流程：
1. 写出当前我当时的判断、理由、动作。
2. 写出专家标杆会如何判断。
3. 写出升维我在用户现实约束下会如何判断。
4. 提炼唯一关键差距。
5. 生成一个下次可执行动作。

### 2. 知识内化

适用：用户说“用这本书/论文/文章训练我”。

不要输出读书报告。必须输出：
- 这份材料解决用户哪个重复短板。
- 哪些观点可转成用户的判断规则。
- 哪些观点不适合用户当前约束。
- 一个新场景迁移题。
- 一个现实执行任务。

### 3. 专家人格对照

适用：用户让某个专家人格、投资大师、产品经理、科学家视角参与训练。

专家只做标杆，不做最终答案。输出必须经过用户模型过滤：
- 保留什么专家原则。
- 删除什么不适配用户约束的建议。
- 转成什么升维我动作。

### 4. 反事实复盘

适用：用户要复盘亏损、返工、错误决策、错过机会。

结构：
```text
当时动作 -> 实际结果 -> 可控变量 -> 不可控噪声 -> 如果按升维我策略会怎样 -> 下次动作门禁
```

### 5. 迁移测试

适用：判断训练是否真的有效。

用一个未见过的新场景测试：
- 是否能调用新规则。
- 是否能说明适用边界。
- 是否能识别反证条件。
- 是否能给出可执行动作。

## 内化评分

不要只给“模型回答质量分”。必须区分：

| 维度 | 问题 |
|---|---|
| 理解 | 用户是否能复述核心机制，而不是复述结论 |
| 迁移 | 用户是否能把规则用到新场景 |
| 决策改善 | 同类问题中判断是否更稳、更少遗漏关键变量 |
| 执行 | 用户是否真的按训练动作做了 |
| 保持 | 延迟几天或几周后是否还保留该动作 |

总分只能代表“内化候选程度”，不能直接代表真实人格变化。

## 回流门禁

```text
训练输出
  -> profiles/*.md 候选升维我
  -> training-log.jsonl 训练摘要
  -> internalization-log.jsonl 内化评分
  -> persona-causal-evolver 判断证据等级
  -> 2-3 个独立现实证据命中
  -> 才能建议回灌真实人格 Skill
```

### 允许写入 profiles

- 专家原则被用户约束过滤后形成的候选策略。
- 训练题、检查清单、反模式和下一步动作。
- 明确标注为“候选/目标态/未验证”。

### 不允许直接写入真实人格

- 单次训练后的高分版自己。
- 只有 AI 输出、没有用户执行证据的结论。
- 一次情绪、一次自动化、一次读书总结。
- 没有反证条件的规则。

## 场景示例

| 场景 | 专家标杆 | 升维我 | 训练任务 |
|---|---|---|---|
| 高风险决策看到利好 | 分析结构、预期差、资源路径 | 结合用户约束、风险承受力和易兴奋点，先写风险和反证 | 行动前写 3 条反证和退出条件 |
| AI Coding 故障 | 给排查路径 | 先找真相源、复现、风险边界，再小修验证 | 每次修复前写 evidence/verdict/blocker |
| 读一本书 | 总结核心观点 | 判断它会修改用户哪条决策规则 | 用一个真实问题应用书中规则 |
| 团队管理 | 授权、激励、绩效 | 让团队下次自己会，而不是 TL 兜底 | 把一次救火转成门禁或模板 |
| 产品判断 | 用户价值和 MVP | 抑制过度系统化，压成最小验证 | 写一个可验收 MVP 假设 |

## 输出要求

每轮训练必须给：
1. 当前我判断。
2. 专家标杆。
3. 升维我判断。
4. 核心差距。
5. 一个训练动作。
6. 内化验证方式。
7. 是否交给 persona-causal-evolver 的建议。
